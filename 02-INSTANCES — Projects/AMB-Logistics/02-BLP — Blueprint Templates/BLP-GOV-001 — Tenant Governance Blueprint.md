# 2. Naming Standard (Deterministic)

### 2.1 Technical Code Structure
Siguiendo el estándar `CAT-SUB-001`, todos los objetos técnicos se nombrarán bajo la lógica:
*   **Users (UPN):** `nombre.apellido@amblogistics.be`
*   **Admins (UPN):** `admin.nombre@amblogistics.be`
*   **Teams:** `TEAM-<DEPT-CODE>`
*   **Groups:** `GRP-<DEPT-CODE>-<FUNCTION>`
*   **Shared Mailboxes:** `alias@amblogistics.be` (Display Name: `MBX - <DEPT-CODE> - <Function>`)

# 3. Ownership Model
Todo objeto (Team, Grupo, Mailbox) debe tener un **Primary Owner** explícito asignado en la fase de Matrix (MTX).
*   **Regla:** Los Shared Mailboxes operativos son propiedad del Jefe de Departamento correspondiente.
*   **Responsabilidad:** El Owner es responsable de la revisión trimestral de accesos.

---

# 4. Identity Lifecycle
1.  **Joiner:** Creación de UPN + Asignación de Licencia + Membresía automática a Team departamental.
2.  **Mover:** Revisión de permisos en 24h tras cambio de departamento.
3.  **Leaver:** Bloqueo inmediato + Conversión de buzón a Shared (si aplica) + Retención de datos según política.

---

# 5. Minimal Admin Sprawl
*   **Global Admins:** Máximo 2 cuentas (una principal, una break-glass).
*   **Delegación:** Uso de roles específicos (User Admin, Teams Admin) en lugar de Global Admin para tareas diarias.
*   **Separación:** Las cuentas admin NO tienen buzón de correo personal; se usan exclusivamente para gestión.
